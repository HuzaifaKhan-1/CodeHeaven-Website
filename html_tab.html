HTML Reference

Our HTML reference provides an in-depth look at every element, attribute, event, and other essential aspects of HTML. This section will serve as your go-to resource for understanding how to use HTML effectively.

HTML Elements

HTML elements are the building blocks of web pages. They define the structure and content of a web page. Below are some of the most commonly used HTML elements:

  • <html>: The root element that wraps all other elements in an HTML document.
  • <head>: Contains metadata about the document, including links to stylesheets and scripts.
  • <title>: Sets the title of the document, which appears in the browser's title bar or tab.
  • <body>: Contains the content of the document, such as text, images, and other media.
  • <h1> to <h6>: Header elements used to define headings, with <h1> being the most important.
  • <p>: Defines a paragraph of text.
  • <a>: Defines a hyperlink to another page or resource.
  • <img>: Embeds an image in the document.
  • <ul>, <ol>, <li>: Define unordered and ordered lists, and list items.
  • <div>: A generic container for grouping and styling content.
  • <span>: A container for in-line content, often used for styling specific text.

HTML Attributes

Attributes provide additional information about HTML elements. They are always included in the opening tag and usually consist of a name and a value.

  • id: A unique identifier for the element, used for styling and accessing elements via JavaScript.
  • class: Specifies one or more class names for an element, which can be used for CSS styling.
  • style: Adds inline CSS to an element for specific styling.
  • href: Specifies the URL for a hyperlink.
  • src: Specifies the source file for embedded content, such as images or videos.
  • alt: Provides alternative text for images, used by screen readers and when images fail to load.
  • title: Offers additional information about the element, often displayed as a tooltip on hover.
  • data-*: Custom attributes used to store extra data on HTML elements, accessible via JavaScript.

HTML Forms

Forms are a crucial part of HTML, used for collecting user input. The following elements are typically used in forms:

  • <form>: Defines the start of a form, containing form controls like input fields.
  • <input>: An essential form element used for user input. Different types of inputs include text, password, email, radio, and checkbox.
  • <label>: Associates a text label with a form control.
  • <textarea>: Allows for multi-line text input.
  • <button>: Defines a clickable button, typically used for form submission.
  • <select> and <option>: Create a drop-down list with various options for the user to choose from.
  • <fieldset> and <legend>: Group related form controls and labels with a caption.

HTML Global Attributes

Global attributes are attributes that can be applied to any HTML element. They are used for configuring various aspects of an element:

  • accesskey: Specifies a shortcut key to activate/focus an element.
  • contenteditable: Indicates whether the content of an element is editable.
  • dir: Specifies the text direction for the content in an element (e.g., "ltr" for left-to-right or "rtl" for right-to-left).
  • draggable: Indicates whether the element is draggable.
  • hidden: Specifies that an element is not yet, or is no longer, relevant.
  • lang: Defines the language of the element's content.
  • tabindex: Specifies the tab order of an element when the user navigates with the keyboard.
  • title: Contains additional information about the element, often displayed as a tooltip when the mouse hovers over it.

HTML Events

HTML events are "things" that happen to HTML elements. These events can be handled using event attributes or JavaScript. Some common events include:

  • onclick: Triggered when an element is clicked.
  • onmouseover: Triggered when the mouse pointer is moved over an element.
  • onmouseout: Triggered when the mouse pointer is moved away from an element.
  • onchange: Triggered when the value of an input element is changed.
  • onsubmit: Triggered when a form is submitted.
  • onload: Triggered when an element has finished loading.
  • onfocus: Triggered when an element gains focus.
  • onblur: Triggered when an element loses focus.

HTML Entities

HTML entities are used to display reserved characters in HTML. They are also used to display characters that are not present on a standard keyboard:

  • &amp;: Displays an ampersand (&).
  • &lt;: Displays a less than sign (<).
  • &gt;: Displays a greater than sign (>).
  • &quot;: Displays a double quotation mark (").
  • &apos;: Displays an apostrophe (').
  • &nbsp;: Inserts a non-breaking space.

HTML Semantic Elements

Semantic HTML elements clearly describe their meaning in a human- and machine-readable way:

  • <header>: Represents a container for introductory content or navigational links.
  • <nav>: Represents a section of the page that links to other pages or to parts within the page.
  • <section>: Defines sections in a document, such as chapters, headers, footers, or any other sections of the document.
  • <article>: Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.
  • <aside>: Represents a piece of content that is only slightly related to the rest of the page content.
  • <footer>: Represents the footer for its nearest sectioning content or sectioning root element.
